
# 堅牢なWebシステム開発 要件および手順書（完全統合版）
**【システム構成】** HTML/CSS/JS, Supabase, GitHub, Resend, GitHub Copilot, LINE API, Google Cloud API

## 1. 開発事前準備（ツール・鍵・コスト）

### ① 必要なツール・ライブラリ
* **開発環境:** Visual Studio Code (VS Code) + GitHub Copilot (AIコーディングアシスタント)
* **バージョン管理・レビュー:** Git, GitHub, CodeRabbit（またはGitHub Actions + CodeQL）
* **セキュリティ・バックエンド機能:** DOMPurify (XSSサニタイズ), Supabase Edge Functions (外部API通信用の安全な裏側環境)
* **外部連携ライブラリ:** `googleapis` (Deno環境からGoogleの各サービスを操作するため)

### ② シークレット（鍵）管理リスト
以下の情報は絶対にネット上（公開リポジトリやフロントエンドのJSコード内）に漏らしてはいけません。`.env`ファイルとGitHub Secrets等で厳重に管理します。

| サービス名 (用途) | 管理すべき重要な情報（シークレット） | 共有・保管のルール |
| :--- | :--- | :--- |
| **ドメイン管理会社** | アカウントのログインID / パスワード | 開発責任者のみ。MFA必須 |
| **GitHub** | リポジトリの管理者権限 | チームメンバーに権限付与 |
| **Supabase** | `Project URL` & `anon key` | フロントエンド(JS)に公開してOK |
| | **`service_role key` (全権限)** | **絶対秘匿。**バックエンドのみで使用 |
| **Resend** | **`API Key`** | **絶対秘匿。**Supabase設定用 |
| **LINE Developers** | **`Channel ID` / `Secret` / `Token`** | **絶対秘匿。**ログイン/通知用 |
| **Google Cloud (GCP)** | **`OAuth Client ID / Secret`** | **絶対秘匿。**Googleログイン用 |
| | **`Service Account JSON Key`** | **絶対秘匿。**システムからGoogleツールを自動操作する為の最も重要な鍵。 |

## 2. 開発ロードマップ（全7フェーズ）

### Phase 0: 各種アカウント・ドメイン・外部窓口の開設
すべての土台となる住所と、各種サービスの裏側（API）を開通させます。
* **【素人向け】** システムの「看板（ドメイン）」を購入し、LINEの公式アカウントと、システムが自動で書き込む先となる「Googleのロボットアカウント」を準備します。
* **【プロ向け詳細】**
    * DNS管理権限の確保。
    * LINE Developersで「LINEログイン」「Messaging API」チャネルを作成。
    * GCPでプロジェクト作成。Calendar, Sheets, Docs, Gmailの各APIを有効化。
    * GCPでOAuth同意画面の設定と、自動処理用の「サービスアカウント（JSONキー）」を発行。

### Phase 1: 安全な開発環境とルールの構築（GitHub）
AIを活用した「ミスを防ぐ仕組み」を作ります。
* **【素人向け】** 本番システムへの直接書き込みを禁止し、AI検査官と人間が二重でセキュリティの穴を確認する「関所」を作ります。
* **【プロ向け詳細】**
    * `main` ブランチの直接Push禁止（ブランチプロテクション）。
    * PR作成とApprove必須化。CI/CD自動レビュー（CodeRabbit/CodeQL）の導入。
    * `Secret Scanning` と `Dependabot` の有効化。環境変数の `.gitignore` 徹底。

### Phase 2: ドメインとメール転送の信頼性設定（Resend + DNS）
システムからのメールが「なりすまし」と判定されない証明を設定します。
* **【素人向け】** ドメインの裏側に証明書を登録し、システムからのメールが迷惑メールに入るのを防ぎます。
* **【プロ向け詳細】**
    * Resendにドメイン登録。DNSに **TXT（SPF, DKIM）**、**MXレコード**、**DMARCレコード**を追加し、ドメイン認証を完了。

### Phase 3: データベースと権限のセキュア設計（Supabase + 外部連携口）
データの金庫とアクセスルールを固め、ログイン用のパイプを繋ぎます。
* **【素人向け】** データベース自体に「一般人は自分のデータのみ」「管理者は全て」のルールを設定します。
* **【プロ向け詳細】**
    * RBACの実装（`user_roles` テーブル、`is_admin()` 関数の定義）。
    * 全テーブルでRLS（行単位セキュリティ）を有効化し、厳密なポリシーを設定。
    * DB側でのバリデーション（`CHECK`制約等）設定。
    * Supabase Authで LINE・Googleプロバイダーを有効化し、APIキーを登録。

### Phase 4: 認証・会員登録の防護壁実装（Frontend）
システムへの「入り口」を作り、外部からの攻撃や不正登録を防ぎます。
* **【素人向け】** スパム攻撃を防ぎ、「LINE/Googleでログイン」ボタンを設置してパスワード不要の安全な入り口を作ります。
* **【プロ向け詳細】**
    * `Email Confirmations` の必須化、レートリミットの設定。
    * User Enumeration防止のためのエラーメッセージ汎用化。OTP有効期限短縮。
    * JSでOAuth2.0フロー（`signInWithOAuth`）を実装。

### Phase 5: 画面・機能開発と自動化ロボット（Edge Functions）の実装
機能を作り込みつつ、各外部ツールとの自動連携を開発する、システムの心臓部です。
* **【素人向け】** 画面を作りつつ、システム裏側のロボットが「LINE通知」「Googleカレンダー登録（Meet発行）」「スプレッドシート記帳」「ドキュメント作成」「Gmail送信」をすべて自動で行うようにプログラミングします。
* **【プロ向け詳細】**
    * フロントエンド防衛: ルートガードの実装。`textContent`や`DOMPurify`によるXSS対策。
    * **Supabase Edge Functions (Deno)の構築:** シークレットを含む外部API通信はすべてここに隔離。
        * LINE API: プッシュメッセージ送信処理。
        * Google API (`googleapis`使用): JSONキーで認証し、`calendar.events.insert` (Meet付与), `sheets.spreadsheets.values.append`, `docs.documents.batchUpdate`, `gmail.users.messages.send` 等を要件に合わせて実装。

### Phase 6: ホスティング（本番公開）とネットワーク保護
完成したシステムを公開し、安全な通信のみを許可します。
* **【素人向け】** 看板（ドメイン）をシステムに繋ぎ、見知らぬ怪しいサイトとの通信を完全に遮断して公開します。
* **【プロ向け詳細】**
    * GitHub Pages/Vercel等へデプロイし、DNSでA/CNAMEレコードを紐づけ。
    * CSP（コンテンツセキュリティポリシー）の設定: 自社ドメイン、Supabase、Resend、LINE API、Google API 以外の通信をブラウザレベルでブロック。
    * Supabaseの `Site URL` と `Redirect URLs` を本番環境に固定し、オープンリダイレクト脆弱性を防止。
